# Network-explorer
# Библиотеки
npm init -y
npm install express multer
Чтобы запустить описанный веб‑проводник в локальной сети, выполните **пошаговую инструкцию** ниже.


---

### 1. Подготовьте окружение

**Требуется:**
- Компьютер с установленным **Node.js** (скачать с [nodejs.org](https://nodejs.org)).
- Базовый текстовый редактор (VS Code, Notepad++, Sublime Text).


**Проверьте установку Node.js:**
```bash
node -v
npm -v
```
Если команды не выполняются — установите Node.js.


---

### 2. Создайте проектную структуру

1. Создайте папку для проекта (например, `network-explorer`).
2. Внутри неё создайте подпапки и файлы:
   ```
   network-explorer/
   ├── public/
   │   └── index.html      # Фронтенд (код из предыдущего ответа)
   ├── uploads/            # Папка для хранения файлов (создайте пустую)
   └── server.js          # Серверная часть (код из предыдущего ответа)
   ```

---

### 3. Установите зависимости

1. Откройте терминал в папке проекта (`network-explorer`).
2. Выполните команду:
   ```bash
   npm install express multer
   ```
   Это установит необходимые пакеты:
   - `express` — веб‑сервер;
   - `multer` — обработка загружаемых файлов.

---

### 4. Настройте фронтенд

1. В файле `public/index.html` найдите строку:
   ```javascript
   const BASE_URL = 'http://ваш-сервер:3000';
   ```
2. Замените `'http://ваш-сервер:3000'` на:
   - Если запускаете на своём ПК: `'http://localhost:3000'`.
   - Если доступ нужен с других устройств в сети: `'http://<ваш-IP>:3000'` (например, `'http://192.168.1.100:3000'`).

   **Как узнать свой IP:**
   - Windows: `ipconfig` в командной строке.
   - macOS/Linux: `ifconfig` или `ip addr`.

---

### 5. Запустите сервер

1. В терминале (в папке проекта) выполните:
   ```bash
   node server.js
   ```
2. В консоли появится сообщение:
   ```
   Сервер запущен на http://localhost:3000
   ```

---

### 6. Откройте проводник в браузере

1. **На том же компьютере:**
   - Откройте браузер и перейдите по адресу:  
     `http://localhost:3000`

2. **С другого устройства в той же сети:**
   - Введите в браузере:  
     `http://<ваш-IP-адрес>:3000`  
     (например, `http://192.168.1.100:3000`).

---

### 7. Проверьте работу

1. Вы увидите интерфейс проводника с пустой папкой `uploads/`.
2. Попробуйте:
   - Загрузить файл через форму.
   - Кликнуть на папку (если создадите её через сервер).

---

### 8. Дополнительные настройки (по желанию)

#### Чтобы сервер был доступен всегда:
- Не закрывайте терминал, где запущен `node server.js`.
- Для фоновой работы используйте `pm2` (установите через `npm install -g pm2`, затем `pm2 start server.js`).

#### Чтобы изменить порт:
- В `server.js` замените `const PORT = 3000;` на другой (например, `8080`).
- При доступе указывайте новый порт: `http://localhost:8080`.


#### Для безопасности (минимум):
- Добавьте базовую аутентификацию (пример для Express: пакет `express-basic-auth`).
- Ограничьте доступ к папке `uploads/` через настройки сервера.

---

### Возможные проблемы и решения


1. **«Cannot get /»**  
   - Проверьте, что файл `index.html` лежит в папке `public/`.
   - Убедитесь, что сервер запущен командой `node server.js`.

2. **«Connection refused»**  
   - Проверьте, что порт (по умолчанию `3000`) не занят.
   - Убедитесь, что IP‑адрес указан верно.

3. **Файлы не загружаются**  
   - Проверьте, что папка `uploads/` существует и у Node.js есть права на запись.

4. **Доступ с других устройств не работает**  
   - Убедитесь, что компьютеры в одной сети (например, подключены к одному роутеру).
   - Проверьте настройки брандмауэра (разрешите входящие соединения на порт `3000`).

---

### Итог

Вы запустили:
- Веб‑интерфейс проводника на HTML/JS.
- Сервер на Node.js, хранящий файлы в папке `uploads/`.
- Доступ по локальной сети (через IP‑адрес).


Для продакшена добавьте аутентификацию и HTTPS!
